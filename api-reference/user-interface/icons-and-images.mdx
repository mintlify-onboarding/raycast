## Types

### Image

Display different types of images, including network images or bundled assets.

Apply image transforms to the source, such as a `mask` or a `tintColor`.

Tip: Suffix your local assets with `@dark` to automatically provide a dark theme option, eg: `icon.png` and `icon@dark.png`.

#### Example

```
// Built-in icon
const icon = Icon.Eye;

// Built-in icon with tint color
const tintedIcon = { source: Icon.Bubble, tintColor: Color.Red };

// Bundled asset with circular mask
const avatar = { source: "avatar.png", mask: ImageMask.Circle };

// Implicit theme-aware icon
// with 'icon.png' and 'icon@dark.png' in the `assets` folder
const icon = "icon.png";

// Explicit theme-aware icon
const icon = { source: { light: "https://example.com/icon-light.png", dark: "https://example.com/icon-dark.png" } };
```

#### Properties

| Property  | Description                                                                                                                    | Type                                                                              |
| --------- | ------------------------------------------------------------------------------------------------------------------------------ | --------------------------------------------------------------------------------- |
| source\*  | The [Image.Source](/api-reference/user-interface/icons-and-images#image.source) of the image.                                  | [Image.Source](/api-reference/user-interface/icons-and-images#image.source)     |
| fallback  | [Image.Fallback](/api-reference/user-interface/icons-and-images#image.fallback) image, in case source can't be loaded.        | [Image.Fallback](/api-reference/user-interface/icons-and-images#image.fallback) |
| mask      | A [Image.Mask](/api-reference/user-interface/icons-and-images#image.mask) to apply to the image.                               | [Image.Mask](/api-reference/user-interface/icons-and-images#image.mask)         |
| tintColor | A [Color.ColorLike](/api-reference/user-interface/colors#color.colorlike) to tint all the non-transparent pixels of the image. | [Color.ColorLike](/api-reference/user-interface/colors#color.colorlike)         |

### Image.Source

```
Image.Source: URL | Asset | Icon | { light: URL | Asset; dark: URL | Asset }
```

The source of an [Image](/api-reference/user-interface/icons-and-images#image). Can be either a remote URL, a local file resource, a built-in [Icon](/api-reference/user-interface/icons-and-images#icon) or a single emoji.

For consistency, it's best to use the built-in [Icon](/api-reference/user-interface/icons-and-images#icon) in lists, the Action Panel, and other places. If a specific icon isn't built-in, you can reference custom ones from the `assets` folder of the extension by file name, e.g. `my-icon.png`. Alternatively, you can reference an absolute HTTPS URL that points to an image or use an emoji. You can also specify different remote or local assets for light and dark theme.

#### Example

```
import { Icon, List } from "@raycast/api";

export default function Command() {
  return (
      
    
  );
}
```

### Image.Fallback

```
Image.Fallback: Asset | Icon | { light: Asset; dark: Asset }
```

A fallback [Image](/api-reference/user-interface/icons-and-images#image) that will be displayed in case the source image cannot be loaded. Can be either a local file resource, a built-in [Icon](/api-reference/user-interface/icons-and-images#icon), a single emoji, or a theme-aware asset. Any specified `mask` or `tintColor` will also apply to the fallback image.

#### Example

```
import { List } from "@raycast/api";

export default function Command() {
  return (
      
    
  );
}
```
